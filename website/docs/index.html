Winmux documentation

Winmux is a desktop terminal workspace for Windows. It keeps related shells together, supports split layouts, and tells you when background work needs attention.

Preview softwareWinmux is under active development. Expect the interface and settings to evolve.
01

Installation

Until packaged downloads are published, run Winmux from the project source.

Requirements

  • Windows 10 or 11
  • Node.js and npm
  • Rust stable with the MSVC toolchain
  • Microsoft Edge WebView2

Run the development build

# from the Winmux project directory
npm install
npm run tauri dev

Create a release build

npm run tauri build

The Windows installer is written to src-tauri/target/release/bundle/.

02

Folders

Folders are independent workspaces that contain terminal tabs and their pane layouts. Use one per project, environment, or area of focus.

  1. Select + beside Folders, or press Ctrl + Shift + N.
  2. Name the folder.
  3. Create terminals inside it. Winmux keeps each folder's sessions separate.

Folders can be renamed, duplicated, reordered, or deleted from their context menu.

03

Shell profiles

Winmux detects common Windows shells and includes profiles for PowerShell, Command Prompt, WSL, and Git Bash when available.

Open Settings → Profiles to choose the default shell or add a custom profile. A profile can set:

  • Executable command and arguments
  • Starting directory and environment variables
  • Display name, icon, and menu visibility
  • Per-profile activity overrides
04

Tabs & panes

A terminal tab starts with one pane. Split it right or down to run related commands side by side. Drag the divider to resize, and use Alt + arrow keys to move focus.

Split rightCtrl ⇧ D
Split downCtrl ⇧ E
New terminalCtrl ⇧ T
Close paneCtrl ⇧ W

Closing a pane with a foreground process asks for confirmation by default.

05

Activity & notifications

Winmux tracks long-running commands and marks panes as completed, failed, waiting for input, or ringing the terminal bell. Native Windows notifications can surface meaningful activity while the app or pane is unfocused.

Privacy by defaultShell integration is optional and never edits your shell profile files.

Configure command-duration thresholds, waiting detection, bell behavior, and notification types under Settings → Activity and Settings → Notifications.

07

Customization

Open Settings → Appearance to choose a theme, accent color, font family, font size, line height, cursor style, and scrollback limit. Changes preview live before you save.

08

Troubleshooting

A shell is not listed

Confirm its executable is installed and available on your system. You can also add it manually under Settings → Profiles.

The desktop app does not start

Install or repair Microsoft Edge WebView2, then run npm run tauri dev again and inspect the terminal output.

Notifications do not appear

Enable native notifications in Winmux and confirm Windows notifications are allowed for the app.
